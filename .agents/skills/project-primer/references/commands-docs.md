# Commands Docs

Read this when adding, renaming, or removing autoloaded public Fish commands under `home/dot_config/fish/functions/`, or when editing `docs/src/commands/` or `docs/.vitepress/commands.sidebar.mts`.

## Contract

Update the VitePress Commands docs in the same change as the Fish function.

## Files to update

1. **`docs/src/commands/<topic>.md`** — add or update a `## \`command\`` section with **Platforms**, a short description, and usage or flags when non-obvious
2. **`docs/src/commands/index.md`** — add or update the topic row in **Browse by topic**; keep rows alphabetically sorted by topic name
3. **`docs/.vitepress/commands.sidebar.mts`** — add a topic link only when creating a new topic page; keep entries alphabetically sorted after `Overview`

Hand-maintain `commands.sidebar.mts`; it is not generated by `docs/scripts/generate-docs.mjs`.

## Platforms

Derive **Platforms** from the function source path:

| Source path | **Platforms** value |
| --- | --- |
| `functions/<name>.fish` | All |
| `functions/unix/<name>.fish` | Unix (`linux`, `darwin`, `wsl`) |
| `functions/darwin/<name>.fish` | macOS (`darwin`) |
| `functions/linux/<name>.fish` | Linux (`linux`) |

On mixed-topic pages, set **Platforms** on each `## \`command\`` section. A single **Platforms** line at the top is enough when every command on the page shares the same scope.

## Topic choice

Prefer an existing topic before adding a new `.md` file:

`apm`, `cloudflare-warp`, `git`, `gnome`, `homebrew`, `mlflow`, `shell`, `toolchains`, `xcode`

## Exclusions

- Private helpers (`__*`) do not need Commands docs unless user-visible behavior changes
- Completion-only changes do not need Commands docs unless usage or flags change

## Verification

```bash
fish --no-execute home/dot_config/fish/relative/path/to/script.fish
npm run docs:lint --prefix docs
```

Run `npm run docs:build --prefix docs` when changing sidebar config or many command pages.
