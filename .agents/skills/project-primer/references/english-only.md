# English-Only Content

Read this before editing stored text in the repository: docs, comments, descriptions, command output, error messages, prompts, and user-facing strings.

## Rule

- Use English for repository content, even when the chat with the user is not in English.
- Keep comments, docs, CLI output, descriptions, errors, and generated text in English.
- Do not add Chinese or other non-English text to code, scripts, docs, or skill files unless an exception applies.

## Exceptions

- Intentionally localized user-facing names or UI strings may stay localized when the surrounding file requires them.
- Files under `rime/` may use Chinese or other locale-specific text.
- When unsure whether an exception applies, use English.
